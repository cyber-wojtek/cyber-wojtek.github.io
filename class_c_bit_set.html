const uint64_t & begin,

inlineconstexprnoexcept

!!! W.I.P. - Does not function correctly at the moment !!!
Fill the bits in the specified range with the specified value using an optimized algorithm.
This algorithm is particularly efficient when the step size is relatively low.
Note: This function has a rather complex implementation. It is not recommended to use it when simple filling without a step is possible.
Performance of this function varies significantly depending on the step. It performs best when step is a multiple of m_chunk_size, and is within reasonable range from it.
However, worst when step is not aligned with m_chunk_size and end is not aligned with m_chunk_size. In such cases, extra processing is required to handle the boundary chunks.

beginBegin of the range to fill (bit index)
endEnd of the range to fill (bit index)
